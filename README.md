# SQL Data Analysis using MySQL

## Overview

This project demonstrates SQL-based data analysis on an E-commerce Sales dataset using MySQL. It covers essential SQL concepts such as data retrieval, filtering, sorting, aggregation, joins, subqueries, views, and indexing to generate meaningful business insights.

## Tools & Technologies

- MySQL
- MySQL Workbench
- SQL

## Dataset

The dataset contains sales transaction details with the following fields:

- OrderID
- Date
- City
- Category
- Product
- Quantity
- Price
- Revenue

## SQL Concepts Covered

- SELECT
- WHERE
- ORDER BY
- GROUP BY
- Aggregate Functions (SUM, AVG)
- INNER JOIN
- LEFT JOIN
- RIGHT JOIN
- Subqueries
- Views
- Indexes

## Queries Performed

- Displayed all records from the dataset.
- Filtered records using the WHERE clause.
- Sorted records using ORDER BY.
- Calculated total revenue by city using SUM().
- Calculated average price by category using AVG().
- Retrieved products with above-average revenue using a subquery.
- Created and queried a SQL View (`City_Revenue`).
- Created a secondary `Products` table and performed:
  - INNER JOIN
  - LEFT JOIN
  - RIGHT JOIN
- Created an index to improve query performance.

## Business Insights

- Identified the total revenue generated by each city.
- Compared average product prices across categories.
- Found products generating above-average revenue.
- Demonstrated how joins combine data from multiple tables.
- Improved query performance using indexing.
